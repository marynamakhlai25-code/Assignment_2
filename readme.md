# Practical assignment 2

Інфраструктура соціальної мережі (Користувачі > Пости > Лайки)
social_users: 100,000 рядків (Дані облікових записів та їхній статус)
social_posts: 1,000 рядків (пости)
social_likes: 1,000,000 рядків (Відстеження активності)

Неоптимізований запит (Підзапити):
Час планування | 14.815 ms 
Час виконання | 370.191 ms

Оптимізований запит (CTE + Віконні функції):
Час планування | 11.912 ms
Час виконання | 166.674 ms

Додаткове завдання (Вимкнені індекси):
Час планування | 5.828 ms
Час виконання | 117.061 ms

Explanation & Terminology

1. **Проблема неоптимізованого варіанту:** 
Базовий запит змушений обчислювати один і той самий набір об'єднаних таблиць чотири рази окремо всередині вкладених підзапитів для знаходження MIN(cnt) та MAX(cnt)
2. **Рішення через CTE та Віконні функції:** 
Використання конструкції WITH (CTE) дозволило виконати JOIN та фільтрацію даних лише один раз. Віконна функція ROW_NUMBER() дозволила відсортувати пости за кількістю лайків за один прохід, скоротивши час виконання з 370 ms до 166 ms
3. **Поведінка оптимізатора (Бонусне завдання):** 
Після виконання команд SET enable_indexscan = OFF планувальник PostgreSQL перестав оцінювати вартість індексних шляхів, що знизило час планування до 5.828 ms.
